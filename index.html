[ PLUGIN ]

Icon categorization

Three categories, three color strategies.

Outlined

Stroke-based. Uses currentColor so your className cascades.

Filled

Solid fill with currentColor. For emphasis states.

Colored

Preserves original palette. For illustrative icons.

Naming validation

PascalCase + duplicates + empty SVG detection. Errors in Chinese.

[ OUTPUT ]

Generated code

forwardRef components, TypeScript, zero config.

React component

forwardRef + IconProps: size / color / strokeWidth.

Types file

Shared IconProps extending SVGProps<SVGSVGElement>.

Barrel exports

Auto-generated index.ts for each category + root.

Atomic commit

All files pushed in a single commit via Git Data API.

[ WORKFLOW ]

How it works

+ + + +
01

Structure your Figma file

Create a page with "icon" in its name. Add top-level frames ending with Outlined, Filled, or Colored. Name components in PascalCase.

02

Configure GitHub connection

Paste your Personal Access Token, target repo, branch, and base path. Stored locally in Figma clientStorage.

03

Scan & Publish

The plugin validates structure, generates React TSX components with proper types, and pushes everything in a single commit.

[ USAGE ]

Use it like Lucide. Or replace Lucide.

Generated components follow the React forwardRef pattern with full TypeScript support.

Your App .tsx 
import { Outlined, Filled, Colored } from '@/icons';

<Outlined.ArrowUp size={20} color="#FF0000" />
<Filled.Heart className="text-red-500" />
<Colored.Flag size={32} />
